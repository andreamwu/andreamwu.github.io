配置Github/Gitee ssh key

文档地址

  • 生成ssh key ：ssh-keygen -t rsa -b 4096 -C "your_email@example.com" 设置名称为github_rsa

  • 创建第二个账号：设置名称为gitee_rsa

  • ssh key 到ssh-agent：

    默认只读取id_rsa，为了让ssh识别新的私钥，需将其添加到ssh-agent中。启动 ssh-agent

$ eval "$(ssh-agent -s)"

$ ssh-add ~/.ssh/id_gitee_ras

  • 配置config：将ssh key key 配置到各自的地址上

  • 配置到GitHub上：

    • GitHub添加ssh key：打开id_github_ras.pub文件，复制内容

    • Gitee添加ssh key：打开id_gitee_ras.pub文件，复制内容

本地配置多个SSH key

  • 新建config文件：touch config

    config的配置没有找到文档说明

    # GitHub
Host github.com
    HostName github.com
    PreferredAuthentications publickey
    IdentityFile ~/.ssh/id_github_ras
  
# Gitee
Host gitee.com
    HostName gitee.com
    PreferredAuthentications publickey
    User anjunwu
    IdentityFile ~/.ssh/id_gitee_ras
# User: username 是对应仓库的用户名

同步Gitee和Github

  • 仓库导入：登录Gitee点击右上角➕导入Github仓库即可

  • 同步更新：

    • 方式一：适合比较少分支的仓库

      • 通过 git remote -v 查看要同步的仓库的远程库列表，如果在列表中没有码云的远程库地址，需要新增一个地址git remote add 远程库名 远程库地址

        git remote add gitee https://gitee.com/andreamwu/hello-git.git
# 将本地Github项目推到Gitee
$ git push gitee master

        若代码提交到Github同操作

    • 方式二：强制更新

      • 如果仅仅同步Github的项目，在Gitee具体仓库界面，仓库名直接点强制同步（新代码会覆盖旧代码）

picGo + Typora + Gitee 搭建个人图床

  • pigGo

    picGo文档

    1. 安装picGo，设置PicGo用Gitee作为图床，搜索插件github-plus安装

    2. Gitee建立公开库images ,过后再建私人令牌，来允许PicGo访问并更新images,私人令牌勾选 user_info projects

    3. 在图床设置中，配置 picGo中的插件 github-plus ，填写repo 为 andreamwu/images ,粘贴 token，origin选择gitee即可。配置完成，可以在picGo的 上传区 上传图片测试，在 相册 中进行管理

    4. 测试结果

  • Typora

    1. Typora → 文件 → 偏好设置 (或者快捷键 ctrl + ,)

    2. 图像，选择插入图片时 → 上传图片，勾选对网络位置的图片应用上述规则,选择上传服务 :PicGo(app)，点击验证图片上传选项。

    3. 验证结果出错了，pigGo无法解析 C:\Program Files 这路径，重新安装pigGo到默认没有文件夹名有空格的路径

    4. 验证出错：Failed to fetch，修改 PigGo设置的端口号，和验证的一致：picGo设置 → 设置server

    5. 验证图片上传选项结果
